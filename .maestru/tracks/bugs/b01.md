# B01: Corrigir lógica de cor da probabilidade de chuva no ItemMobile

## Problema

Em `src/components/itemMobile/ItemMobile.js`, a lógica de cor do `probRainColor` tem dois `if` com a mesma condição (`< 30`), o que faz com que o estado "amarelo" (probabilidade média) nunca seja atingido.

```js
// Código atual — BUG
if (props.value.precipitaProb < 30) {
  probRainColor = "rgb(34, 199, 34)";
} else if (props.value.precipitaProb < 30) {  // ← nunca verdadeiro
  probRainColor = "darkyellow";
}
```

## Solução

Alinhar com a lógica correta usada em `ItemBrowser.js`:

```js
if (props.value.precipitaProb <= 30) {
  probRainColor = "rgb(34, 199, 34)";
} else if (props.value.precipitaProb < 69) {
  probRainColor = "darkyellow";
} else {
  probRainColor = "red";
}
```
