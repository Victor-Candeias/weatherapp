# B05: Adicionar tratamento de erro visível ao utilizador

## Problema

O hook `useHttp` (`src/hooks/use-http.js`) captura erros HTTP e guarda-os no estado `error`, mas nenhum componente apresenta esse erro ao utilizador. Se a API do IPMA falhar, o utilizador vê um estado de "Loading" infinito sem qualquer mensagem.

## Solução

Em `App.js` e `Body.js`, verificar o estado `error` retornado pelo hook e renderizar uma mensagem de erro adequada:

```jsx
// Exemplo em App.js
const { isLoading, error, sendRequest: fetchTasks } = useHttp();

if (error) {
  return <div className={classes.error}>Erro ao carregar dados: {error}</div>;
}
```

Aplicar o mesmo padrão em `Body.js` para o fetch da previsão da cidade.

## Ficheiros

- `src/components/App.js`
- `src/components/body/Body.js`
- `src/hooks/use-http.js` (verificar se `error` é devolvido corretamente)
